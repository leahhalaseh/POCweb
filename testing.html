Testing Plan

Testing Strategy

Category Why Who What When Where How
Unit Testing To make sure code is written correctly Developers All code written As soon as new code is written Local Dev Automated, JUnit, TestNG, PHPUnit
API/Service Testing To ensure communication between components are working Developers New web sites, components, controllers, etc. When new API is developed and ready Local Dev Automated, Soap UI, Rest Client
Acceptance Testing To ensure customer's expectations are met Developer/SDET/Manual QA Verifying acceptance tests on the stories, verification of features When the feature is ready and unit tested CI/Test Environment Automated (Cucumber)
System Testing / Regression Testing / UAT To ensure the whole system works when integrated SDET/Manual QA/Business Analyst/Product Owner Scenario testing, user flows and typical user journeys, performance and security testing When acceptance testing is completed Staging Environment Automated (Webdriver) exploratory testing
